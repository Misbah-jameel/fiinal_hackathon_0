# CEO Briefing Skill

## Trigger
Use this skill when the user runs `/ceo-briefing` or asks for a "CEO briefing", "weekly briefing", "business audit", or "Monday morning briefing".

## Purpose
Generate a comprehensive **Monday Morning CEO Briefing** that includes:
1. Financial summary (Odoo accounting data if available)
2. Task/project status
3. Email/communication summary
4. Social media performance
5. Pending approvals
6. Alerts and action items

## Steps

### 1. Read Vault State
- Read `AI_Employee_Vault/Dashboard.md`
- List all files in `AI_Employee_Vault/Needs_Action/`
- List all files in `AI_Employee_Vault/Pending_Approval/`
- List all files in `AI_Employee_Vault/Plans/`
- Read `AI_Employee_Vault/Company_Handbook.md`

### 2. Gather Financial Data (if Odoo available)
- Call `odoo_get_balance` to get receivables/payables
- Call `odoo_get_revenue` for current month revenue
- Call `odoo_list_payments` for recent payments (last 7 days)
- If Odoo unavailable: note "Odoo offline — manual review needed"

### 3. Gather Social Analytics (if configured)
- Call `social_get_analytics` for all platforms
- Summarize follower counts, recent engagement

### 4. Read Recent Logs
- Read `AI_Employee_Vault/Logs/` — last 7 days of log files
- Identify key events, errors, completions

### 5. Generate Briefing
Create a briefing file at:
`AI_Employee_Vault/Briefings/YYYY-MM-DD_Monday_Briefing.md`

The briefing should include:

```markdown
---
type: ceo_briefing
generated: <ISO timestamp>
period: <date range>
---

[2-3 sentence summary of the week]

[list from /Plans/]

[list from /Done/ with dates]

[list from /Needs_Action/]
```

### 6. Update Dashboard
- Update `AI_Employee_Vault/Dashboard.md` with briefing generated timestamp
- Add briefing to Recent Activity

### 7. Log Action
- Append to `AI_Employee_Vault/Logs/YYYY-MM-DD.md`:
  `- [timestamp] CEO Briefing generated → Briefings/YYYY-MM-DD_Monday_Briefing.md`

### 8. Notify Owner
- Print summary to console
- Mention the briefing file location

## Notes
- If Odoo MCP is not running, skip financial section and note it
- If social credentials are missing, note which platforms are unconfigured
- Always generate the briefing even if some data sources are unavailable
- Mark unavailable data clearly with "⚠️ Data unavailable"
